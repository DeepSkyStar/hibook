# Hibook

Hibook 是一套定义了 **Markdown 结构规范** 并能将其快速转换成网站预览及导出为 PDF/HTML 电子书的自动化工具集。

## 目录结构规范
要让一个目录成为标准的 "Hibook" 项目（使其支持 `hibook web` / `hibook export`），它必须遵循以下约定：

1. **`SUMMARY.md`**：这是整本书籍/知识库的索引文件，用于定声明文档树。格式如下：
   *   根级别链接为一级目录；
   *   每缩进一次（2个空格或Tab）为次级目录；
   *   所有链接的路径均为相对路径。
   *   示例：
        ```markdown
        * [第一章 前言](README.md)
        * [第二章 核心理论](01-core/theory.md)
          * [2.1 模型](01-core/model.md)
        ```

2. **`README.md`**: 作为索引的进入点或书籍的总体介绍。
3. **媒体和资源文件**: 推荐统一放置在指定的目录，或与 md 文件同级，在 md 中使用正常的 Markdown 相对路径引用，不能引用工作区外的绝对路径。
4. **内部链接**：若要在文档间互相引用，必须严格使用 Markdown 链接格式。例如 `[参考第二章](../01-core/theory.md)`。不能只写文本路径。当生成 PDF 或电子书时，标准的内部相对链接会自动转化为章节页内的内部锚定跳转（Anchor）。

## Hibook 命令行工具
`hibook` CLI 工具可通过全局注册（或局部命令）向遵循上述规范的知识库进行一键部署和导出。

### `hibook web`
*   **功能**：在当前目录下启动一个简单的本地 Web Server（默认端口 `3000`）。
*   **用途**：用于在写作时快速进行双屏预览或检查文件修改的结果。使用 `Cmd+R` 在浏览器中手动刷新即可看到最新变更内容。

### `hibook export`
*   **功能**：读取当前工作目录下的 `SUMMARY.md` 文件，根据里面标定的结构顺序，将所有 Markdown 文件依序全量合并，并解析出内置的数学公式（MathJax 支持）以及 Mermaid 图表。
*   **输出**：它将会在当前目录的 `export/` 文件夹下生成一份 `knowledge_full.md`（方便继续导入其他发布渠道）以及一份支持打印的 `knowledge_full.html` 文件。
*   **PDF导出**：只需在浏览器（推荐 Chrome/Safari）中打开 `export/knowledge_full.html`，右键选择“打印”（Print），另存为 PDF（建议开启背景图形以保留样式）即可生成格式优美的电子书或者文档集。
