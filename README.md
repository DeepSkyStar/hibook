# Hibook

Hibook 是一套极其轻量、却又拥有重型特性的 **极客个人知识库系统**与**自动化工具集**。它彻底摒弃了传统的复杂数据库绑定与私有格式，严格基于纯文本 Markdown 结构与底层 Git 协议，为您提供如 Obsidian 般的本地知识管理体验。

> **🗺️ 核心架构**：基于纯粹的 Git 仓库与 Markdown 树展开。所有的内部连结、知识图谱（Graph View）、全文检索均在内存或本地构建侧动态生成。详情请参考 **[项目架构规约 (ARCH.md)](./ARCH.md)**。

## 核心特性
1. **Global Hub Daemon (全局多路复用中枢)**：抛弃每个文档库占一个独立端口的旧模式。只需启动一次后台守护进程 (`hibook start -p 3000`)，您即可在一个极具现代感的聚合 Web Dashboard 中统一挂载、管理并无缝切换多个独立的知识库工程。
2. **交互式全息阅读器**：内置基于 Docsify 的高级皮肤引擎，原生支持 MathJax 数学公式和 Mermaid 内联图表，以及侧边栏文档目录树的隐式路由。
3. **集成化编辑器 (WYSIWYG)**：提供本地实时双屏 Markdown 编辑能力，所有修改可结合后端的 `/_api/save` 接口直接物理落地并静默 Commit 到目标 Git 工程中。
4. **知识图谱与全局检索 (Graph & Search)**：自动化构建每个知识库内部相互依赖节点的拓扑图（Graph View），并通过本地后台的 SQLite 索引模块提供毫秒级的代码级全局全文检索。
5. **CLI 工具链**：内置了一键式的生成、排版验证（`update`）以及纯静态脱机打包导出为 PDF/HTML 电子书的服务（`export`）。

## 目录结构规范
要让一个目录成为标准的 "Hibook" 项目空间，它必须遵循以下约定：
1. **`SUMMARY.md`**：整本书籍/知识库的索引文件，用于声明左侧边栏文档树。根级别为一级目录；每缩进一次为次级目录。
2. **`README.md`**: 作为索引的初始进入面。
3. **`RULE.md`**: 放置当前知识库收纳与书写的约束性规约（`hibook create` 时将自动模板化注入）。
4. **内部链接**：若要在文档间互相引用，请使用标准 Markdown 相对路径链接（如 `[参考](../01-core/theory.md)`）。当使用编辑器或导出时，标准相对链接会自动映射为底层动态路由。

## CLI 命令行参考
`hibook` 的核心服务已经全面向守护进程（Daemon）模型演进。

| 命令 | 详细作用 |
| :--- | :--- |
| `hibook start [-p 3000]` | 在后台 3000 端口启动基于 多路复用 (Multiplexing) 架构的 **全局 Hub Dashboard**，并自动托管在浏览器呈现。 |
| `hibook stop` | 平滑下线后台托管中枢服务。 |
| `hibook create [dir]` | 在指定的目录下快速 Scaffold (脚手架架构) 一个初始化完毕的 Hibook 模板，并自动建立底层的 Git 追踪。 |
| `hibook update` | 依据物理文件夹架构树的变动，智能化自动续写/调整 `SUMMARY.md`。 |
| `hibook export` | 解析指定的索引树结构，完整合并为单独的 `knowledge_full.html` / `knowledge_full.md` 静态排版，方便打印 PDF 或注入其它系统。|

*(更多详细架构隔离机制与前端底层规约，见 `ARCH.md`)*
